# **🎵 Ion Music Box 开发文档 (v1.0)**

本项目是一款基于 **Tauri v2** 构建的极轻量级桌面音乐播放器。它采用 Rust 处理底层音频解码与文件系统，使用原生 JavaScript 编写极简 UI，并支持通过自定义 JS 脚本扩展网络音源。

## **1\. 技术栈选型**

* **内核框架**: [Tauri v2](https://v2.tauri.app/) (高性能、低内存、原生窗口)  
* **后端语言**: Rust (负责文件扫描、ID3 解析、音频解码、网络代理)  
* **前端语言**: Vanilla JavaScript (无框架，极致轻量)  
* **核心库 (Rust)**:  
  * lofty: 音频元数据 (ID3, FLAC tags) 解析。  
  * rodio: 跨平台音频播放引擎。  
  * walkdir: 高效递归文件扫描。  
  * reqwest: 处理音源脚本发起的跨域请求。

## **2\. 核心架构设计**

项目采用 **“宿主-沙盒”** 架构：

1. **宿主 (Rust)**: 提供受信任的 API，如读取本地文件、控制播放器硬件。  
2. **UI 层 (WebView)**: 负责界面渲染。  
3. **插件沙盒 (Web Worker)**: 运行用户导入的 JS 音源脚本，通过桥接 API 与宿主通信。

## **3\. 核心功能实现方案**

### **3.1 本地音乐扫描 (已初步实现)**

通过 walkdir 遍历目录，利用 lofty 提取 title 和 artist。

* **优化点**: 封面图采用延迟加载。Rust 端不返回 Base64，而是提供一个自定义协议 URL（如 asset://localhost/path/to/music.mp3），由 Rust 在读取时实时提取封面流。

### **3.2 自定义 JS 音源兼容层 (核心)**

为了运行您提供的 lx 脚本，我们需要在前端构建一个模拟环境。

**实现思路：**

1. **环境注入**: 在 Web Worker 或隐藏的 iframe 中定义 globalThis.lx。  
2. **API 映射**:  
   * lx.request: 映射到 Rust 端的 proxy\_http\_request 命令，绕过浏览器 CORS 限制。  
   * lx.send(EVENT\_NAMES.inited): 触发 UI 更新，解锁搜索功能。  
3. **音源获取**: 脚本返回 URL 后，前端将其传递给 Rust 的播放引擎。

### **3.3 低资源音频播放 (Rust rodio)**

不使用 HTML5 \<audio\> 标签，以降低内存占用。

* **流程**: 前端传 path 或 url 给 Rust ![][image1] Rust 创建 Decoder ![][image1] 送入 Sink 播放。  
* **优势**: 支持全局媒体按键控制，即使窗口最小化也极其稳定。

### **4.2 JS 音源桥接逻辑 (sandbox.js)**

// 模拟 LX 音乐的环境  
const setupLxSandbox \= (scriptContent) \=\> {  
  globalThis.lx \= {  
    EVENT\_NAMES: { request: 'request', inited: 'inited' /\* ... \*/ },  
    request: async (url, opts, callback) \=\> {  
      // 调用 Rust 端的网络代理  
      const resp \= await window.\_\_TAURI\_\_.core.invoke('http\_proxy', { url, opts });  
      callback(null, resp);  
    },  
    send: (name, data) \=\> {  
      if (name \=== 'inited') unlockOnlineFeatures(data.sources);  
    }  
  };  
  // 执行音源脚本  
  eval(scriptContent);  
};

## **7\. 注意事项**

1. **路径处理**: Windows 路径需注意转义 (D:\\\\Music)。  
2. **内存占用**: 避免在前端存储大量歌曲元数据，建议只存储当前显示的部分。  
3. **安全**: 执行 eval() 运行音源脚本时，确保只在受控环境下进行，不暴露敏感系统 API。
